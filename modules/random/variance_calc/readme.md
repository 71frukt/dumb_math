## О катастрофической потере точности
Результат вычитания двух (положительных) приблизительно равных чисел много менее аккуратен, чем операнды:

$\tilde{x} = x (1 + \sigma_x)$, $\tilde{y} = y (1 + \sigma_y)$, где $\sigma_x$ и $\sigma_y$ - относительные ошибки представления $x$ и $y$

Сами ошибки $\sigma_x$ и $\sigma_y$ связаны с результатами предыдущих вычислений, измерениями, дискретизацией, моделированием.

$\tilde{x} - \tilde{y} = (x - y) \left(1 + \frac{x\sigma_x - y\sigma_y}{x - y}\right)$,

если $x$ и $y$ приблизительно равны, то ошибка результата вычитания нарастает.

Рассмотрим конкретный пример, когда ошибка вычислений возрастает катастрофически сильно.


## Mетоды расчёта дисперсии

Рассмотрим методы вычисления смещенной оценки дисперсии $DX = E(X - EX)^2, DX \ge 0$, на базе выборки $(x_1, \dots, x_n)$

* Быстрый: $\quad\quad\quad\quad DX = RS - M^2$, где $RS = \frac{1}{n} \sum_{i=1}^n x_i^2$, $M = \frac{1}{n} \sum_{i=1}^n x_i$
* Двухпроходной: $\quad M = \frac{1}{n} \sum_{i=1}^n x_i$, $DX = \frac{1}{n} \sum_{i=1}^n (x_i - M)^2$
* Однопроходной: $\quad M_{n+1} = M_n + \frac{x_{n+1} - M_n}{n}$, $DX_{n+1} = DX_n + \frac{(x_{n+1} - M_n)(x_{n+1} - M_{n+1}) - DX_n}{n}$

Теоретически Быстрый метод должен давать наибольшую погрешность при выборке с большим средним и малом среднеквадратичном отклонении, так как он основан на вычитании двух близких друг другу больших чисел. Оценить сразу, какой - Двухпроходной или Однопроходной метод - окажется точнее, сложно, поэтому сделаем вывод на основе эксперимента. 


## Размеры типов и диапазоны значений на тестирующей машине
Диапазоны значений определим исходя из стандарта *IEEE 754*. В нормализованном виде число вычисляется по формуле:
$$Value = (-1)^S \cdot (1 + M) \cdot 2^{E - B}$$

Максимальное нормализованное число - все биты экспоненты равны 1, кроме младшего (значение - 254), все биты мантиссы равны 1.

Минимальное денормализованное число (ближайшее к нулю) - поле экспоненты $E = 0$, неявная единица пропадает, формула меняется на $0.M \cdot 2^{1 - B}$. Ближайшее к нулю ненулевое число получается, когда в мантиссе стоит единственная единица в самом младшем разряде.
$$V_{min\_denorm} = 2^{-m} \cdot 2^{1 - B} = 2^{1 - B - m}$$

### Разбор особенностей
1. Алгоритм *OnePass* выдает нулевую дисперсию для $\texttt{float}$, начиная с mid = 10000.

Рассмотрим случай с mid = 10000. Определим $\texttt{ULP}$ (Unit in the Last Place) для чисел типа $\texttt{float}$ порядка $10000_{10}$. Как было написано выше, нормализованное число представляется как $V = (-1)^S \cdot 1.M \cdot 2^{E_{true}}$.  
$2^{13} = 8192$  
$2^{14} = 16384$  
Следовательно, истинная экспонента $E_{true} = 13$. Длина интервала $\Delta = 2^{14} - 2^{13} = 8192$.

float: мантисса $m = 23$ бита $\implies C = 2^{23}$ возможных комбинаций.  
double: мантисса $m = 52$ бита $\implies C = 2^{52}$ возможных комбинаций.

$\texttt{ULP}_{float, 10^{5}} = \frac{\Delta}{C} = 0.0009765625$

$\texttt{ULP}_{double, 10^{5}} = \frac{\Delta}{C} \approx 1.818989 \times 10^{-12}$


Заметим, что
$\texttt{ULP}_ {float, 10^{5}} \approx 0.0009$ > $0.0001$, 
то есть слагаемые суммы 
$\sum_ {i=1}^n (x_i - M)^2$
не представимы и обращаются в $10000$, дисперсия выборки из одинаковых элементов — нуль.


2. Катастрофическая потеря точности Быстрого алгоритма для $mid = 10^7$ и больше.

Алгоритм вычисляет суммы  
$RS = \frac{1}{n} \sum_{i=1}^n x_i^2$, $M = \frac{1}{n} \sum_{i=1}^n x_i$.  
Как было описано выше, $\texttt{ULP}$ растёт с увеличением представляемого числа. Сумма квадратов $x_i^2$ выборки из 1000 элементов c $mid=10^7$ имеет порядок $10^3 \times 10^{7 \times 2} = 10^{17}$.  
$\texttt{ULP}_{float, 10^{17}} = 2^{\lfloor\log_2 10^{17}\rfloor - 23} = 2^{56 - 23} \approx 10^{10}$

Огромное значение $\texttt{ULP}$ позволяет качественно объяснить возникновение огромной погрешности Быстрого алгоритма для выборки с большим средним и малой дисперсией.
