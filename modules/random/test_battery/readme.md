## О тестировании ГСЧ
Фактически этот проект является библиотекой для тестирования произвольных генераторов случайных чисел. В качестве примера будет протестирован написанный в прошлом задании генератор $\chi^2$.

## Принцип тестирования
Тестирование любого ГСЧ сводится к тестированию разными методами некоторой последовательности, которую выдает генератор. Для возможности проверки ГСЧ разными тестами, используя один инструментарий, используется метод двухуровневого тестирования:
* Имеется последовательность из $N$ чисел на отрезке $[0, 1]$ - $\{x_j\}$. Она разбивается на последовательность из $k$ блоков - $\{block_i\}$.
* Строится тест, как-то сопоставляющий входному блоку $\{block_i\}$ число $Y_i$ - тестовую статистику.
* Этот тест должен знать, какую функцию распределения $F(Y)$ имеет статистика для гипотетического идеального генератора. $F(Y_i)$ называют **$p\text{-value}$ первого уровня** — вероятность получить такой же или более экстремальный результат при условии, что генератор идеален.
* $\{F(Y)\}$, а значит и $\{F(Y_i)\}$, должны иметь равномерное распределение, что алгоритму и остаётся проверить тестом Колмогорова-Смирнова или Андерсона-Дарлинга, вернув **итоговое $p\text{-value}$ второго уровня**.

Метод удобен тем, что является инструментарием для любого теста, поскольку сводит тестирование к проверке распределения на равномерность.


Возвращаемое значение метода запуска теста есть смысл лишь оценивать, а точнее смотреть, насколько оно близко к 0 или к 1. То есть любые значения результата, лежащие в $[ 0 + \alpha; 1 - \alpha]$ можно считать индикатором качества генератора. Как правило $\alpha$ выбирается порядка $0.02$.

## Тесты
Тестирование проводилось с использованием генератора случайных чисел с равномерным распределением $\texttt{mrg32k3a}$, не считается эталонным.

В проекте реализованы два метода определения тестовой статистики - тест непересекающихся векторов и спектральный тест на основе дискретного преобразования Фурье.

### О спектральном анализе Фурье
Спектральный тест (на основе алгоритма FFT) применяется не к непрерывным случайным величинам на интервале $[0, 1]$, а непосредственно к битовому потоку, который генерирует базовый источник энтропии. Для этого каждое 32-битное состояние генератора разбивается на отдельные биты, которые отображаются в массив значений $\{-1.0, 1.0\}$. 

Тест ищет периодические паттерны в полученной последовательности бит, раскладывая последовательность по Фурье. Отстутствие доминирующих частот свидетельствует о равномерном распределении битов последовательности.

Разложение по Фурье осуществляется алгоритмом FFT (fast Fourier transform). Валидация алгоритма осуществляется набором тестов:

| Название теста | Входные данные | Ожидаемый результат (FFT) |
| :--- | :--- | :--- |
| **Единичный импульс**<br>(Impulse Response) | `[1, 0, 0, ..., 0]`<br>*(Дельта-функция)* | `[1, 1, 1, ..., 1]` |
| **Постоянная составляющая**<br>(DC Component) | `[1, 1, 1, ..., 1]`<br>*(Прямая линия)* | `[N, 0, 0, ..., 0]` |
| **Чистая гармоника**<br>(Single Harmonic) | Синусоида с частотой `k` | Два пика амплитудой `N/2` на позициях `k` и `N-k`. Остальное — `0`. |

## Пример плохого генератора, проходящего тест KS
Этот генератор обладает идеальными статистическими показателями распределения, но полностью лишен случайности:
$$x_n = \frac{n \pmod M}{M}$$
где $M$ — количество шагов (размер периода). При больших значениях $M$ (например, $10^4$) последовательность имитирует непрерывное равномерное распределение $U(0, 1)$.

Генератор так хорошо проходит тест KS по нескольким причинам:
* Инвариантность к перестановкам: критерий KS вначале своей работы сортирует элементы, то есть работает с ней как с множеством элементов.
* Функция распределения представляет собой идеальную ступенчатую аппроксимацию прямой $F(x) = x$. Максимальное отклонение $D_n$ составляет $\approx \frac{1}{M}$, что при $M=1000$ дает $D_n = 0.001$, гарантируя $p\text{-value} \approx 1.0$.
* Математическое ожидание последовательности на интервале $[0, 1]$ с шагом $1/M$ в точности совпадает с теоретическим значением $0.5$ (с погрешностью $O(1/M)$).

на выходе мы получим:
```zsh
KS Statistic (Dn): 0.001
P-Value:           1
```

То есть генератор идеален. Очевидно, что этот генератор не пройдёт второй уровень тестирования - мы получим $p-value_2 = 0$, так как выборка {$p-value$} не будет равномерно распределенной на [0, 1]. Поймём, что идеальное распределение было "идеально" в каждом тесте, то есть генератор слишком идеален, чтобы быть случайным.
